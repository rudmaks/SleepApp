1
Подключить Groq

Шаг 3 — Git
4
Инициализировать репозиторий
/undo и /redo работают только в git-репозитории:
cd sh_toolkits
git init
git add .
git commit -m "init"
5
Добавить .gitignore
data/reports/
Исключает папку reports/ — AI не будет читать логи без явного запроса.

--- description: Создать новый .sh скрипт по шаблону проекта --- Создай новый скрипт scripts/$ARGUMENTS.sh для Android мониторинга. Соблюдай архитектуру проекта из AGENTS.md. Скрипт независимый, без меню, отчёт в: ~/storage/shared/scripts/reports/ГГГГ-ММ-ДД_ЧЧ-ММ_$ARGUMENTS.txt

Старт сессии
1
Открыть проект
cd ~/sh_toolkits
opencode
AGENTS.md и schema.md подхватываются автоматически.
2
Обновить текущую задачу
Перед началом — обновить строку ## Текущая задача в AGENTS.md.
Цикл работы со скриптом
A
Разработка
AI читает и редактирует scripts/имя.sh напрямую. Не создаёт дубли — только редактирует существующий файл.
B
Отправить на устройство
LocalSend → Internal storage/scripts/ → на устройстве:
cp ~/storage/shared/scripts/имя.sh ~/.shortcuts/
chmod +x ~/.shortcuts/имя.sh
C
Получить данные обратно
LocalSend: отчёт или data.js → sh_toolkits/data/, затем:
/datacheck
AI подтверждает timestamp и готов к анализу.
D
Показать лог напрямую в чате
!cat data/reports/2026-04-10_cpu.txt
Вывод добавляется в контекст как результат инструмента.
Управление контекстом
!
Контекст заполнился
/compact
Сжимает историю, сохраняя суть. Или /new для чистого старта.
!
Отменить изменения файла
/undo
Откатывает последнее сообщение и все изменения файлов. Требует git.
Агенты
B
Build — по умолчанию
Пишет и редактирует файлы. Основной режим разработки.
P
Plan — переключить через Tab
Только анализ без изменений файлов. Использовать когда нужно обсудить архитектуру или проверить логику скрипта.
